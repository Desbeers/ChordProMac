This is an unsupported arm version of ChordPro for macs running on Apple Silicon

Distributing ‘ready to go’ binaries for Apple Silicon is a lot harder than on Intel. The application must be signed with a valid Apple Developers account or else it will simply not run.

I don’t have such account.

The version in the Resources folder is ‘ad-hoc’ signed; that means its signed to run on my mac and my mac only. Sorry, you cannot run it.

If you want to use it; you have to ‘re-sign’ it so it becomes yours. The Install.zsh can help you.

To use this script; the Xcode Command Line Tools must be installed. Fortunately; they are free and do not require any account. A full Xcode is not required. Use your favourite search engine if you don’t know how to install it.

Once you have the Xcode Command Line Tools installed, you can run the Install.zsh script in the Terminal. It does require your administration password; so use it at your own risk!

It will do the following:

  • Copy ChordPro to your applications folder
  • Remove all attributes that prevents code-signing
  • Re-sign the application ‘ad-hoc’, so its just for you
  • Remove the quarantine bit so macOS will not show any warning when opening the first time
  • Add the chordpro command-line command to your $PATH so you can easily use it in the Terminal to use its full power

This is dirty; I know, but it works...
